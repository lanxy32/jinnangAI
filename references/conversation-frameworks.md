# 销售对话推进框架

把这个文件当成“怎么组织一句话”的总控台。

## 一、总节奏：A -> B -> P -> D

### A 起点唤醒
让客户看到：现在不动，其实有代价。

### B 未来投射
让客户看到：做了之后，状态会怎么变。

### P 路径框定
让客户感觉：开始并不难，第一步很轻。

### D 动力激发
让客户觉得：现在行动，比继续拖更划算。

## 二、12 个话术模块

### 1. Warm Opening
适合开场降防备。

### 2. Inverse Framing
适合打破旧认知。

### 3. Empathy Matching
适合先接情绪。

### 4. Desire Priming
适合激发动机。

### 5. Opportunity Framing
适合把“犹豫”改写成“机会窗口”。

### 6. Micro Commitment
适合降低行动门槛。

### 7. Progressive Optioning
适合用双选推动，而不是空问“你考虑一下吗”。

### 8. Scenario Bridging
适合画面化未来结果。

### 9. Third-party Proofing
适合建立信任，但不要编造案例。

### 10. Trend Anchoring
适合借行业趋势增强判断感。

### 11. Problem Reveal
适合让客户自己意识到问题。

### 12. Action Urgency
适合推动当下动作，但不要捏造稀缺。

## 四、三种训练重点的推荐组合

### 异议处理
推荐顺序：
共情 -> 反问/重构 -> 低门槛体验 -> 下一步动作

常用组合：
- Empathy Matching + Inverse Framing + Micro Commitment
- 情感同频法 + 成本对比法 + 引导体验法

### 快速成交
推荐顺序：
问题揭示 -> 结果画面 -> 双选推进 -> 行动窗口

常用组合：
- Problem Reveal + Progressive Optioning + Action Urgency
- 未来对照法 + 双选推动法 + 即效承诺法

### 高效共情
推荐顺序：
示弱开场 -> 接感受 -> 理解顾虑 -> 给轻动作

常用组合：
- Warm Opening + Empathy Matching + Micro Commitment
- 示弱换信任法 + 情感同频法 + 请教式尊重引导

## 五、每条回复最多叠三层

不要把 5 个模块塞进一条回复。短回复里，最多叠 2 到 3 个动作最有效。

## 六、默认话术骨架

可优先套用这个骨架：

1. 我理解你在担心什么
2. 真正要看的不是表面，而是关键变量是什么
3. 你现在最轻的一步可以怎么做
4. 你做完这一步，再决定要不要继续

## 七、需要避免的错误

- 一上来就反驳客户
- 只讲产品，不讲客户当下顾虑
- 空泛鼓励，没有下一步动作
- 把“体验、试用、预约、复盘”这些低门槛动作漏掉
- 强行制造稀缺和承诺
