# 知识地图

先看这份地图，再决定读什么。

## 默认必守

1. 所有输出先遵守 `sales-principles.md`
2. 所有对话推进优先使用 `conversation-frameworks.md`
3. 所有评分、考核、模拟优先使用 `question-bank-and-scoring.md`

## 按任务读取

### 1. 做销售考核
优先读取：
- `question-bank-and-scoring.md`
- `industry-question-bank.md`
- `report-template.md`

必要时补读：
- `customer-psychology.md`
- `payment-and-value.md`

### 2. 做销售答疑
优先读取：
- `conversation-frameworks.md`
- `question-bank-and-scoring.md`

必要时补读：
- `payment-and-value.md`
- `customer-psychology.md`
- `industry-question-bank.md`

### 3. 做客户角色模拟
优先读取：
- `question-bank-and-scoring.md`
- `industry-question-bank.md`
- `customer-psychology.md`

结束时再读取：
- `report-template.md`

### 4. 做总结 / 生成 PDF
优先读取：
- `report-template.md`

必要时回看：
- `question-bank-and-scoring.md`
- 当前对话里的评分记录

## 各文件作用

- `sales-principles.md`：统一语言风格与表达边界
- `conversation-frameworks.md`：决定每一轮话术怎么推进
- `question-bank-and-scoring.md`：通用异议题库、评分维度、客户类型、示范话术
- `payment-and-value.md`：解释客户为什么愿意买、什么时候该升维、什么时候该筛选客户
- `customer-psychology.md`：用三大中心模型理解客户的深层驱动力
- `industry-question-bank.md`：多行业场景题库；健身最细，其他行业为可直接使用的起步版
- `report-template.md`：总结报告和 PDF 输出模板
